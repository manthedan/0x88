        Use the controls above to paste a FEN and analyze it (analysis is drawn on the board as arrows), choose your side, pick the engine reply mode (policy-only or fixed-visit search), set visits/batch, and clear the model cache. Watch LC0 game plays the LC0 search engine out on the board against LC0 policy or Stockfish lite (adjustable depth) so you can watch the moves; press Stop to end it. The WebGPU row flags when an inference backend silently fell back to WASM. Query params still work for smokes: open with ?ep=wasm&parity=1 to force WASM and automatically compare browser outputs against committed native BLAS FEN-only plus explicit-history fixture priors with elapsed time and eval/s. Use ?search=1 to auto-run a fixed-visit PUCT root search, ?worker=1&search=1 to run search in a worker, or ?workerOnly=1 / ?bigModel=1 to load the LC0 model only inside the dedicated worker and broker all eval/search requests through it. Add ?bench=1&ep=webgpu&benchWarmup=5&benchIters=25 to run a dedicated-worker eval timing harness. Add ?batch=8 with a matching fixed-batch model such as ?model=/models/lc0/t1-256x10-distilled-swa-2432500.batch8.f16.onnx to select/evaluate search leaves in real ONNX batches. Use ?earlyStop=root-dominance, ?earlyStop=best-stable, or ?earlyStop=kld-stable to opt into guarded LC0-style early stop telemetry. Add ?packProbe=1 to load and verify the batch-8 lc0web packed-model shards inside the dedicated worker without creating an ONNX session, ?kernelProbe=1 to run one custom WGSL fixed-shape MatMul+Add kernel against tensors from that pack, or ?kernelBench=1&kernelBenchIters=1000 to queue many WGSL dispatches and read back once for correctness; add ?kernelVariant=tiled16 for the workgroup-reduction kernel or ?kernelVariant=scalar-transposed to benchmark transposed weight layout. Use ?ortOpBench=1&ep=webgpu to compare the same extracted MatMul+Add as a tiny ORT ONNX op, ?qkvProbe=1 to run the encoder0 Q/K/V projection sub-block with per-iteration readback, or ?qkvBench=1&qkvBenchIters=1000 to queue many Q/K/V projection dispatches and read back once. Use ?attentionScoreBench=1 for the encoder0 Q @ Kᵀ * scale WGSL score op, ?attentionScoreOrtBench=1&ep=webgpu for the equivalent tiny ORT score op, ?softmaxBench=1 for the fixed-shape attention softmax probe, ?attentionValueBench=1 for the fixed-shape softmax(QK) @ V value op, ?attentionBlockBench=1 for the combined input → Q/K/V → QK → softmax → attn@V block, or ?attentionOutputBench=1 to include output projection, alpha residual, and ln1. Add ?cache=1 to persist the selected ONNX model in Cache Storage. Use ?model=/models/lc0/t1-256x10-distilled-swa-2432500.batch1.f16.onnx to smoke the f16 deployment artifact.
